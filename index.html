Because of how technology has progressed today, there are a wide variety of people using the internet, apps, websites, and more. However, as people learn to code websites, they also need to learn how to make their website more accessible. It would not be fair to exclude someone from using the website simply because there were no measures put in place for them to be able to interact with the website too. As a result, here you can learn more about some tools to use to combat this issue.


First, it is important to focus on each group you are coding for. What could be an issue for them? How can this issue be solved? Click on the topics below to learn more.


Visually impaired/blind

This photo is from Unsplash. Bruno Cervera. (2021, September 20). A computer mouse and keyboard. https://unsplash.com/photos/a-computer-mouse-and-keyboard-rpLX5uIhrZg

Key things to incorporate:

  • Add alternate (alt) text for images
    • Although screen readers can be very helpful, they cannot literally "read" images. This is something a programmer needs to add to their code so that the screen reader can describe the image. Make sure the alt code is very clear and descriptive of what is shown.
  • Clear labels
    • To make it easier for people to use the website properly, it is important to label everything clearly. For example, if there is a text box, it must have clear instructions for what the text box is there for.
  • Smooth keyboard use
    • Create an efficient order of the website so that pressing tab goes to the next section.
    • Create buttons/links for the user to skip to the main content of the page.
  • Transcriptions
    • For the screen reader to describe content like videos or audio clips, it can be helpful to include a transcript of what is said and some descriptions.

This information is from Lange, A. (2024, March 25). Ensuring Website Accessibility for the Blind: A Comprehensive Guide. A11Y Collective. https://www.a11y-collective.com/blog/blind-website-accessibility/

Color blind

This photo is from Unsplash. Lucas K. (2018, November 8). Blue and orange smoke. https://unsplash.com/photos/blue-and-orange-smoke-wQLAGv4_OYs

There are many different types of color blindness, but the main thing to strive for is color contrast. Here are some key things to incorporate:

  • Contrasting colors
    • It is important to put colors that contrast enough, particularly if it is something important to the website, like a button. Using a contrast checker can help with this.
  • Be somewhat independent from colors
    • Although the colors are important for the design of a website, they do not need to be the only element to use. Adding shapes or other visuals can also help differentiate between certain sections, labels, or headings.

This information is from Lange, A. (2024, March 18). An Introduction to Colour Blindness Accessibility. A11Y Collective. https://www.a11y-collective.com/blog/color-blind-accessibility-guidelines/

Hard of hearing/deaf

This photo is from Unsplash. Kelly Sikkema. (2023, September 16). A close up of four different types of buttons. https://unsplash.com/photos/a-close-up-of-four-different-types-of-buttons-e5I_83RQi58

Key things to incorporate:

  • Add captions to audio elements.
    • The main key is to add captions for audio. Although AI or other technology can automatically generate captions, it can be wrong. It is important to check it or do it yourself.
  • Create good audio/caption controls
    • There needs to be good control buttons for users so that they can press play and know that the video is playing (playing it automatically can cause problems). Also, being able to control when to put the captions is important too.
  • Well-positioned captions
    • The captions provided on the website must appear at the right time to go along with the audio, and they should not block the screen either. Being able to piece together the visual part of the video along with the captions smoothly is important.

This information is from Curtis, J. (2021, September 24). How to Make a Website Accessible for People Who Are Deaf. AudioEye. https://www.audioeye.com/post/how-to-make-a-website-accessible-for-the-deaf/
